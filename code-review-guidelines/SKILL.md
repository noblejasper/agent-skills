# Code Review Guidelines

コードレビューのガイドラインを提供するスキルです。コードの品質、可読性、保守性、セキュリティ、パフォーマンスを向上させるための指針を提供します。

## 使用するとき

- "コードをレビューして"
- "このコードを改善して"
- "コードの品質をチェックして"
- "このコードの問題点を教えて"
- "Review this code"
- "Check code quality"

## レビューの手順

1. **可読性** を確認する
2. **保守性** を確認する
3. **セキュリティ** を確認する
4. **パフォーマンス** を確認する
5. **改善提案** をまとめる

## チェックリスト

### 可読性

- [ ] 変数名と関数名は意味を明確に表しているか
- [ ] 関数は単一責任の原則に従っているか (1関数 = 1つのことをする)
- [ ] 複雑なロジックには適切なコメントがあるか
- [ ] マジックナンバーや文字列リテラルは定数として定義されているか
- [ ] インデントとフォーマットが一貫しているか

### 保守性

- [ ] DRY原則 (Don't Repeat Yourself) に従っているか (コードの重複がないか)
- [ ] SOLID原則が適用されているか
- [ ] 関数の長さは適切か (目安: 20-30行以内)
- [ ] 適切な抽象化がされているか
- [ ] テストが書きやすい構造になっているか

### セキュリティ

- [ ] ユーザー入力は適切に検証・サニタイズされているか
- [ ] 認証・認可は適切に実装されているか
- [ ] センシティブな情報 (パスワード、APIキーなど) がハードコードされていないか
- [ ] SQLインジェクション、XSSなどの脆弱性がないか
- [ ] エラーメッセージが内部情報を漏洩していないか

### パフォーマンス

- [ ] ネストしたループや冗長なデータベースクエリがないか
- [ ] 大きなデータセットに対して適切なアルゴリズムを使用しているか
- [ ] 不要な計算や重複処理がないか
- [ ] メモリリークの可能性がないか
- [ ] 非同期処理が適切に使用されているか

## 出力フォーマット

レビュー結果は以下の形式で提示します:

```
## コードレビュー結果

### 🔴 重要な問題
- 問題の説明と改善案

### 🟡 改善提案
- 提案の説明

### 🟢 良い点
- 良い実装の説明

### まとめ
総合的な評価と優先度の高い改善点
```
