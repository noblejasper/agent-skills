## SOLID原則

| 原則 | 説明 |
|------|------|
| S - 単一責任 | クラスは1つの責任のみを持つべき |
| O - 開放/閉鎖 | 拡張に対して開き、変更に対して閉じる |
| L - リスコフの置換 | 派生クラスは基底クラスと置換可能であるべき |
| I - インターフェース分離 | クライアントが使わないメソッドに依存させない |
| D - 依存性逆転 | 抽象に依存し、具体に依存しない |
